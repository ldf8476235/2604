# 订单中心改造计划

## 目标
把当前个人中心中的“个人中心”模块改造成“订单中心”，支持购买/出售订单切换、状态/时间筛选、订单列表、站内详情层，并同时兼容 PC 与 H5。

## 步骤
1. 盘点现有前后端订单链路与个人中心结构，明确可复用接口、缺失字段和页面入口。
2. 扩展后端订单接口：补充订单中心列表查询（买家/卖家）、筛选参数、详情结构、进度、操作能力字段；保持项目现有 Spring Boot + MyBatis-Plus + 统一日志体系。
3. 改造前端个人中心：将“个人中心”菜单与内容替换为“订单中心”，完成购买/出售订单 Tab、筛选条、列表与详情弹层。
4. 同步 H5 响应式：确保筛选、列表、详情在移动端单列布局，无横向溢出，底部导航不遮挡。
5. 联调与验证：前端 build、后端 compile、浏览器 PC/H5 实测订单中心列表与详情展示。

## 风险与取舍
- 当前项目若缺少完整支付/群聊/售后链路，本轮先把“进入群聊/申请售后/取消订单”做成可控的状态化入口与按钮显示，不强行接不存在的业务。
- 若卖家订单当前数据库字段不足，则优先通过现有 order/listing/auth_user 组合补齐展示字段，避免新增不必要表结构。
