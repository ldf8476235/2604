## 目标

把代肝订单与交易订单中的“联系客服 / 进入群聊”接成真实 IM 会话，支持站内聊天、图片/文件消息、已读标记，并保持 PC/H5 响应式可用。

## 步骤

1. 补齐 IM 数据模型
   - 新增 `im_conversation`、`im_participant`、`im_message`
   - 启动时幂等补表

2. 实现 IM 后端服务
   - 通过 `chat_group_no` 解析来源订单（交易/代肝）
   - 首次进入时自动创建会话、参与人和系统欢迎消息
   - 支持消息查询、发送文本/图片/文件、标记已读
   - 给测试环境增加客服自动回复

3. 接入前端 IM 页面
   - 新增 `/im/:conversationNo` 路由
   - 拆分聊天页：头部会话信息、消息列表、输入区、上传入口
   - 支持 loading / error / empty / success
   - 支持 H5 全屏聊天布局

4. 打通入口
   - 代肝订单管理的“联系客服”跳转 IM
   - 订单中心的“进入群聊”跳转 IM

5. 联调与验证
   - PC / H5 验证聊天页面无横向溢出
   - 验证发送文本、上传图片/文件、已读标记、客服自动回复
