# 我的发布

## 目标

在现有账号发布链路上补齐“我的发布”能力，覆盖列表、详情、编辑、下架、重新提交审核，并保证 PC / H5 一致体验。

## 实施步骤

1. 梳理现有 `account_listing` 模型、发布状态和发布页字段，明确“我的发布”需要的状态流转与可编辑范围。
2. 在后端 listing 模块新增“我的发布”接口：
   - 列表查询
   - 详情查询
   - 编辑保存
   - 下架
   - 重新提交审核
3. 复用现有发布页表单作为编辑页，通过查询详情预填，避免维护第二套发布表单。
4. 新增“我的发布”页面，包含：
   - 状态筛选
   - 列表展示
   - 详情抽屉 / 弹层
   - 操作按钮与禁用规则
5. 在发布页和全局导航补“我的发布”入口，保证 PC / H5 都可达。
6. 完成构建、接口联调与响应式验证，确认状态流转和交互无回归。
